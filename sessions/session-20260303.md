## What I understood
- Window functions allow you to perform calculations across a set of table rows that are somehow related to the current row. They are often used for ranking, cumulative sums, moving averages, and other calculations that require access to multiple rows of data.

- OVER( ) = full table

- ORDER BY in OVER clause = cumulative sum

- The `OVER` clause is used to define the window for the function. It can include partitioning (using `PARTITION BY`) and ordering (using `ORDER BY`) to specify how the rows are grouped and sorted for the calculation.

- ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
