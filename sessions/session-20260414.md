# Session – 2026-04-14

## Topics covered
- Database Indexing Strategies
- Performance optimization techniques for large databases

## What I understood
- Database indexing is a critical technique for improving query performance by allowing the database to quickly locate and access the relevant data without scanning the entire table.
- There are various types of indexes (e.g., B-tree, hash, bitmap) that are suitable for different types of data and query patterns, and choosing the right index type can significantly impact performance.
- Performance optimization techniques include query optimization, proper indexing, and database configuration tuning to ensure that the database can handle large volumes of data efficiently while maintaining fast response times.

## What is still confusing
- The specific scenarios in which certain indexing strategies are more effective than others, especially in the context of complex queries and large datasets.
- The trade-offs between different indexing strategies in terms of storage overhead, maintenance costs, and query performance, particularly when dealing with dynamic data that frequently changes.

## Questions
